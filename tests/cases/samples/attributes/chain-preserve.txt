
PRESERVE CHAIN - CASE 1

This attribute tests the preservation formation of all tags contained in the token.
This structure is one of the more complex ones that might be encountered.




<div

id="foo"
{% if foo %}
data-{% if x %}-foo{% else %}-bar{% endif %}="{{ x }}"
{% endif %}

>
</div>



PRESERVE CHAIN - CASE 2

This is a little simpler that CASE 1 in the sense that liquid tokens are conditionally
passed, whereas CASE 1 uses an infusion approach.

<div

id="foo"
{% if x  %}data-attr="x"
{% elsif %}
data-id="x"
{% endif %}

></div>


PRESERVE CHAIN - CASE 3

This chain is similar to CASE 1 and CASE 2 but we express a structure which will
result in either a boolean attribute type or attribute value.

<div

data-{% if x %}id="foo"{% else %}name{% endif %}

></div>
